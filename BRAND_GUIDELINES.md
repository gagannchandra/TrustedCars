## Logo System

### Primary Logo
The TrustedCars logo consists of:
1. **Icon**: House/shield symbol (trust + home metaphor)
2. **Wordmark**: "TrustedCars" in Manrope Bold
3. **Tagline**: "Premium Pre-Owned" in Inter Medium

```
┌─────────────────────────────┐
│   [🏠]  TrustedCars         │
│        Premium Pre-Owned    │
└─────────────────────────────┘
```

### Logo Variations

#### 1. **Primary Horizontal Lockup** (Default)
- Use: Website header, marketing materials, presentations
- Colors: Icon (Primary Blue #0B3A6E), Text (Slate 900 #0F172A)
- Minimum width: 120px digital, 25mm print

#### 2. **Stacked Lockup** (Vertical)
```
    [🏠]
TrustedCars
Premium Pre-Owned
```
- Use: Square spaces, social media, app icon
- Minimum width: 80px digital, 20mm print

#### 3. **Horizontal Compact** (No tagline)
```
[🏠] TrustedCars
```
- Use: Tight spaces, small screens, email headers

#### 4. **Icon Only**
```
[🏠]
```
- Use: Favicon, app icon, social avatar
- Minimum size: 24px (any smaller = illegible)

#### 5. **Wordmark Only**
```
TrustedCars
```
- Use: When icon is implied, plain text contexts

### Logo Colors

#### Primary (Preferred)
- **Icon**: Primary Blue (#0B3A6E)
- **Wordmark**: Slate 900 (#0F172A)
- **Tagline**: Slate 600 (#475569) at 80% opacity
- **Background**: White or light (Slate 50 #F8FAFC)

#### Reversed (Dark Backgrounds)
- **Icon**: White (#FFFFFF)
- **Wordmark**: White (#FFFFFF)
- **Tagline**: White (#FFFFFF) at 80% opacity
- **Background**: Primary 800+ or dark image

#### Monochrome Dark
- All elements: Slate 900 (#0F172A)
- Use: Print, single-color applications

#### Monochrome Light
- All elements: White (#FFFFFF)
- Use: Dark backgrounds, reversed applications

### Clear Space
- **Rule**: Maintain clear space equal to the height of the icon on all sides
- **Minimum**: Never allow text, graphics, or edges closer than 1x icon height

```
        [clear space]
[clear]   [🏠]TC    [clear]
        [clear space]
```

### Logo Don'ts

❌ **Never:**
1. Stretch or distort proportions
2. Rotate at angles
3. Add effects (drop shadow, glow, bevel)
4. Change colors outside approved palette
5. Rearrange elements
6. Place on busy backgrounds without background box
7. Use gradients on logo (flat colors only)
8. Outline or add stroke
9. Use old/outdated versions
10. Recreate or redraw (always use official files)

### Logo on Photography

When placing logo on images:
1. **Add white box** with 80% opacity behind logo
2. **Or use dark overlay** (40-60% black) on image, then white logo
3. **Or place in clear area** of photo (sky, solid color)
4. Never place directly on busy patterns

## Color Usage Guidelines

### When to Use Each Color

#### Primary Blue (#0B3A6E)
**Use for:**
- Logo icon
- Main navigation
- Section headings
- Primary brand elements
- Links (hover state)
- Brand backgrounds

**Don't use for:**
- Primary CTA buttons (use success green instead)
- Error/warning states
- Body text (too low contrast)

#### Success Green (#10B981)
**Use for:**
- Primary CTA buttons ("Buy", "Sign Up", "Search")
- Success messages
- Checkmarks, verification badges
- Positive status indicators
- "Available", "Approved" states

**Don't use for:**
- Navigation elements
- Body text
- Neutral information

#### Ice Blue (#BAE6FD)
**Use for:**
- Hero headline accents
- Premium badge backgrounds (light tint)
- Subtle hover effects
- Decorative elements
- Section backgrounds (very light #F0F9FF)

**Don't use for:**
- Primary actions
- Body text
- Standalone without context

#### Amber (#F59E0B)
**Use for:**
- Premium/featured badges
- "Hot deal" labels
- Star ratings
- Limited-time offers
- VIP/exclusive markers

**Don't use for:**
- Error messages (use red instead)
- Regular badges
- Primary CTAs

#### Slate (Neutrals)
**Use for:**
- Body text (700)
- Secondary text (600)
- Borders (200-300)
- Backgrounds (50-100)
- Disabled states (400)

**Don't use for:**
- Primary CTAs
- Brand expression
- Attention-grabbing elements

## Typography Guidelines

### Hierarchy System

```
┌─────────────────────────────────────────┐
│ Hero Headline       72px / Extrabold    │  Primary impact
│ Page Title          48px / Bold         │  Main page heading
│ Section Heading     36px / Bold         │  Major sections
│ Subsection          24px / Bold         │  Sub-sections
│ Card Title          20px / Bold         │  Cards, modules
│ Body Large          18px / Medium       │  Emphasized body
│ Body Text           16px / Normal       │  Standard body
│ Secondary Text      14px / Normal       │  Captions, meta
│ Fine Print          12px / Normal       │  Legal, labels
└─────────────────────────────────────────┘
```

### When to Use Each Font

#### Manrope (Display)
**Use for:**
- All headings (h1-h6)
- Buttons and CTAs
- Navigation labels
- Badge text
- Card titles
- Stats and numbers
- Taglines

**Characteristics:**
- Geometric, modern
- Slightly rounded (friendly)
- High x-height (legible)
- Professional but warm

#### Inter (Body)
**Use for:**
- Body text and paragraphs
- Form inputs and labels
- Descriptions
- Lists
- Tables
- Long-form content

**Characteristics:**
- Designed for screens
- Excellent readability
- Neutral, professional
- Open-source

#### JetBrains Mono (Monospace)
**Use for:**
- Code snippets
- VIN numbers
- License plates
- Technical specifications
- Data tables (when alignment matters)

### Text Formatting Rules

#### Line Length
- **Optimal**: 60-75 characters per line
- **Maximum**: 90 characters
- **Use** `max-w-prose` (65ch) for long-form text

#### Line Height
- **Body text**: 1.5 (150%)
- **Headings**: 1.25 (125%)
- **Large display**: 1.1 (110%)
- **UI elements**: 1.0-1.25

#### Paragraph Spacing
- **Between paragraphs**: 1em (16px)
- **After headings**: 0.5-0.75em
- **Before headings**: 1.5-2em (create separation)

## Photography Style

### Do's ✅

1. **Real vehicles, real locations**
   - Actual inventory photos
   - Well-lit, natural lighting
   - Multiple angles
   - Clean, detailed

2. **Lifestyle imagery**
   - Happy customers (with permission)
   - Families, professionals
   - Modern, aspirational
   - Diverse representation

3. **Environment**
   - Clean, uncluttered backgrounds
   - Modern showrooms
   - Well-maintained facilities
   - Urban/suburban settings

4. **Technical quality**
   - High resolution (min 1920px wide)
   - Sharp focus
   - Proper exposure
   - Accurate colors

### Don'ts ❌

1. **Avoid generic stock photos**
   - Overly polished models
   - Obvious studio lighting
   - Unrealistic scenarios
   - Dated imagery

2. **Poor quality**
   - Blurry, pixelated
   - Poor lighting
   - Color casts
   - Distortion

3. **Misleading**
   - Photoshopped beyond reality
   - Hiding vehicle flaws
   - Different car than advertised
   - Fake backgrounds

### Image Treatment

#### Hero Backgrounds
- **Subtle overlays**: 40-60% dark gradient
- **Blur backgrounds**: 20-40px when text overlay
- **Desaturate**: -20% to -40% for backgrounds (let UI color pop)

#### Car Photos
- **No filters**: Show accurate colors
- **Neutral backgrounds**: White, light gray, or natural
- **Consistent lighting**: Avoid harsh shadows
- **Multiple views**: Front, side, interior, details

## Iconography

### Icon Style
- **Stroke-based** (not filled) for most icons
- **2px stroke weight** at default size
- **Rounded corners** (2px radius on strokes)
- **24px base size** (scale up/down as needed)
- **Monochrome** (inherit text color)

### Icon Library
Use **Lucide React** or **Heroicons** for consistency.

## Patterns & Textures

### Approved Patterns

#### 1. **Subtle Grid** (Backgrounds)
- Dot grid pattern
- 60px spacing
- 5% opacity white on dark
- 2% opacity primary on light

#### 2. **Diagonal Lines** (Accents)
- 45° angle
- 100px spacing
- 3% opacity
- Use in hero sections

#### 3. **Noise Texture** (Depth)
- Very subtle grain
- 2-5% opacity
- Adds warmth to flat colors
- Use on large color blocks

### Pattern Usage
- **Maximum 1 pattern per section**
- **Keep opacity low** (2-5%)
- **Use sparingly** (accent, not dominant)
- **Never obscure content**

## UI Elements

### Buttons

#### Hierarchy
1. **Primary**: Green gradient, white text (main action)
2. **Secondary**: White with border (alternative action)
3. **Tertiary**: Text link only (less important)

#### Sizing
- **Large**: 56px height (Hero CTAs)
- **Default**: 48px height (Most buttons)
- **Medium**: 40px height (Compact UIs)
- **Small**: 32px height (Tables, tight spaces)

### Cards

#### Elevation Levels
1. **Flat** (border only): Lists, grids
2. **Level 1** (shadow-sm): Default cards
3. **Level 2** (shadow-md): Hover state
4. **Level 3** (shadow-lg): Featured cards
5. **Level 4** (shadow-xl): Modals, popovers

#### Field States
1. **Default**: Slate 200 border
2. **Focus**: Primary 600 border + ring
3. **Error**: Error 600 border + ring

5. **Disabled**: Slate 100 background, Slate 400 text

## Motion & Animation

### Timing
- **Fast**: 150ms (micro-interactions)
- **Base**: 200ms (default transitions)
- **Slow**: 300ms (cards, panels)
- **Slower**: 400ms (page transitions)

### Easing
- **Default**: `ease` (CSS default)
- **Entrances**: `ease-out` (start fast, slow down)
- **Exits**: `ease-in` (start slow, speed up)
- **Interactive**: `ease-in-out` (smooth both ends)

### Animation Principles

#### 1. **Purpose-Driven**
Every animation should serve a purpose:
- Provide feedback (button press)
- Show relationship (drawer slides from nav)
- Direct attention (success checkmark)
- Add delight (subtle hover effects)

#### 2. **Subtle & Refined**
- Avoid bouncy, exaggerated effects
- Keep transforms small (2-4px lifts)
- Prefer opacity and transform over color changes
- Less is more (premium = restrained)

#### 3. **Performance**
- Use `transform` and `opacity` only (GPU-accelerated)
- Avoid animating `width`, `height`, `top`, `left`
- Use `will-change` sparingly
- Respect `prefers-reduced-motion`

### Common Animations

```css
/* Lift on hover */
hover:-translate-y-1 transition-transform

/* Scale on hover */
hover:scale-[1.02] transition-transform

/* Fade in */
animate-fade-in-up

/* Shimmer loading */
animate-shimmer

/* Button press */
active:translate-y-0 active:scale-[0.98]
```

## Accessibility Standards

### WCAG AA Compliance (Minimum)

#### Color Contrast
- **Normal text** (16px): 4.5:1
- **Large text** (24px+): 3:1
- **UI elements**: 3:1

#### Text Sizes
- **Minimum**: 12px (labels only)
- **Body**: 16px (default)
- **Never** below 12px

#### Touch Targets
- **Minimum**: 44x44px (Apple HIG)
- **Recommended**: 48x48px
- **Spacing**: 8px between targets

#### Keyboard Navigation
- All interactive elements must be keyboard accessible
- Visible focus states required
- Logical tab order
- Skip links for main content

### Screen Reader Support
- Semantic HTML (`<button>`, `<nav>`, `<main>`)
- ARIA labels where needed
- Alt text for images
- Form labels properly associated

### ✅ Do
- Be transparent about everything
- Show real proof (stats, testimonials)

- Make vague claims
- Hide information
- Use aggressive sales tactics
- Overcomplicate processes
- Ignore accessibility
- Use inconsistent branding
- Mislead customers
